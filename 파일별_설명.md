### `eco_input.py` (약 1,700줄)
**역할:** 측정인.kr 사이트에 측정 데이터를 자동으로 입력하는 메인 로직  
> 백데이터 로직은 `backdata_utils.py`, 탭4 로직은 `tab4_utils.py`, 공통 상수는 `measin_constants.py`로 분리됨

#### 진입점
```
main()
  ├─ 매체 선택 (1=대기 / 2=수질)
  ├─ _main_air()   ← 대기 흐름
  └─ _main_water() ← 수질 흐름
```

#### 대기 흐름 `_main_air()`
| 단계 | 함수 | 설명 |
|------|------|------|
| 시료 목록 수집 | `extract_samples_from_nas()` / `parse_sample_input()` | 팀+날짜 자동추출 또는 직접입력 |
| 탭1 | `fill_tab1()` | 현장측정정보 입력 |
| 탭2 | `fill_tab2()`, `rg2_fill_measure_grid_api()` | 시료채취/측정정보 + 리얼그리드 |
| 탭2 PDF | `export_pdf_from_excel()` | 측정기록부 PDF 생성 후 업로드 |
| 백데이터 | `export_backdata_moist_thc()` ← backdata_utils | 수분 CSV + THC CSV/FID 저장 |
| 탭4 | `fill_tab4_dates()`, `fill_tab4_grid_only()` ← tab4_utils | 측정분석결과 입력 |
| 탭4 PDF | `make_tab4_pdfs()` | 대기 시트 PDF 생성/업로드/최종본 저장 |

#### 수질 흐름 `_main_water()`
탭1/2/백데이터 없음. **탭4 PDF 업로드 + 저장만** 수행
```
탭4 클릭 → make_tab4_pdfs_water() → upload_tab4_pdfs() → tab4_comp_save()
```

#### 주요 상수 (파일 상단 74~105줄)
```python
# 대기
FIELD_URL        = "https://측정인.kr/ms/field_outair.do"
NAS_BASE         = r"\\...\2.성적서"
NAS_DIRS         = ["0 0.입력중", "0 1.완료", ...]
PDF_BASE_DIR     = r"\\...\0.PDF\2.대기pdf"
FINAL_DONE_DIR   = r"\\...\0 5.최종완료"
AIR_FILE_BTN1/2  = "#newFile1" / "#newFile2"
AIR_SHEET_ANALY  = "대기시료채취 및 분석일지"
AIR_SHEET_RECORD = "대기측정기록부"
AIR_SHEET_COVER  = "시험항목날짜서명"

# 수질
FIELD_URL_WATER    = "https://측정인.kr/ms/field_outwater.do"
NAS_BASE_WATER     = r"\\...\14.수질성적서"    ← YYYY년\업체명\ 하위
PDF_BASE_DIR_WATER = r"\\...\0.PDF\1.수질pdf"  ← YYYY년\업체명\ 하위
WATER_FILE_BTN1/2  = "#anzeFile1" / "#anzeFile2"
WATER_SHEET_ANALY  = "분석일지"
WATER_SHEET_RECORD = "대행기록부"
```

#### 자주 수정할 함수
| 함수 | 수정 시점 |
|------|----------|
| 상수 블록 | NAS 경로 변경, 사이트 URL 변경, 셀렉터 변경 시 |
| `make_tab4_pdfs()` | 대기 엑셀 시트명 변경 시 |
| `make_tab4_pdfs_water()` | 수질 엑셀 시트명 변경 시 |
| `build_final_paths()` | 대기 PDF 저장 경로 구조 변경 시 |
| `build_final_path_water()` | 수질 PDF 저장 경로 구조 변경 시 |
| `upload_tab4_pdfs()` | 셀렉터 파라미터로 받음 → 대기/수질 둘 다 이 함수 사용 |

### `backdata_utils.py` (약 500줄) — **신규 분리 모듈**
**역할:** 백데이터(수분량·THC) 추출 유틸 — eco_input.py에서 분리

| 함수 | 설명 |
|------|------|
| `export_backdata_moist_thc()` | 수분 CSV + THC CSV/FID 메인 처리 |
| `export_csv_display_as_is()` | 엑셀 표시값 그대로 CSV 저장 |
| `export_fid_by_excel_copy()` | THC PF 시트 → .FID 파일 저장 |
| `_clipboard_clear()` / `_clipboard_get_unicode_text()` | 클립보드 안전 조작 |
| `_copy_range_and_get_text()` | Range.Copy → 클립보드 텍스트 폴링 |
| `_safe_write_file()` | NAS 안전 쓰기 (재시도 로직) |

백데이터 생성 시 항목별 종료시간 전에는 파일을 만들지 않는다.

- 수분: `입력!B23` 시작시간 + 5분 전이면 `⚠ 수분 백데이터 생성 스킵` 로그 출력 후 패스
- THC: `입력(분석값)!K64` 기준으로 FID는 해당 시각, PF는 +3분 전이면 `⚠ THC 백데이터 생성 스킵` 로그 출력 후 패스

상수: `MOIST_ROOT`, `THC_ROOT`, `XL_UP`, `XL_TOLEFT`

#### 주요 기능
- **매체 선택** (대기/수질) UI 동적 전환
- **작업선택 + 모드선택** 같은 줄에 좌우 배치
- **파일 관리 버튼**: 파일추가(엑셀만)·선택삭제·전체비우기 (대기/수질 각각)
- **드래그&드롭**: tkinterdnd2로 엑셀 파일을 드롭하면 시료번호 자동 추출

- pywinauto STA COM 경고 자동 억제

#### 핵심 구조
`fake_input` 패턴: `eco_input.main()`의 `input()` 호출을 가로채서 GUI 입력값을 순서대로 주입

```python
# _main_water() input 순서 → GUI answers 순서
["2", login_id, login_pw, 시료번호, PDF여부]

# _main_air() input 순서 → GUI answers 순서  
["1", job, login_id, login_pw, mode, 탭1, 탭2, PDF, 백데이터, 탭4, PDF최종본, 시료번호or(팀+날짜)]
```

> ⚠️ `eco_input.py`에서 `input()` 호출 순서가 바뀌면 여기 `answers` 배열도 반드시 같이 수정해야 함

#### 매체 선택에 따른 UI 전환
- 대기 선택 → 작업선택/모드/탭1~탭4 체크박스 활성화, 수질 영역 비활성화 (테두리·글씨 흐리게)
- 수질 선택 → 수질 시료번호 입력창 + PDF 체크박스만 활성화

### `file_utils.py` (약 220줄)
**역할:** NAS 파일 검색 + 시료 목록 수집 + 파일명 기준 비산먼지 판정 공통 로직

| 함수 | 설명 |
|------|------|
| `find_excel_for_sample(sample_no, nas_base, nas_dirs, strict)` | NAS에서 시료번호에 맞는 엑셀 파일 재귀 탐색 |
| `find_best_matching_file(sample_no, nas_base, nas_dirs, extensions, strict)` | 확장자 우선순위까지 포함한 공통 파일 선택 엔진 |
| `collect_samples_from_nas(nas_base, nas_dirs, date_str, team_nos, ...)` | 날짜/팀번호 기준으로 시료 목록 수집 |
| `is_fugitive_dust_file(file_name, keywords)` | 파일명에 '비산먼지' 포함 시에만 비산먼지 파일로 판정 |

- `strict=True`: 시료번호 뒤에 숫자가 오는 파일 제외 (eco_check 방식)
- `strict=False`: 접두사 일치면 허용 (eco_input 방식)
- `~$` 임시파일 자동 제외

## 수정 시 체크리스트

### NAS 경로가 바뀌었을 때
- `eco_input.py` 상단 상수 블록 (`NAS_BASE`, `NAS_BASE_WATER` 등)
- `eco_check.py` 상단 상수 블록
- `backdata_utils.py` 상수 (`MOIST_ROOT`, `THC_ROOT`)
- `tab4_utils.py` 상수 (`ANZE_XLSM`)

### 사이트 셀렉터가 바뀌었을 때
- **탭2 필드 셀렉터** → `measin_constants.py`의 `SEL_*` 상수 수정 (양쪽 자동 반영)
- `eco_input.py` 상수 블록 (`AIR_FILE_BTN1/2`, `WATER_FILE_BTN1/2` 등)
- `tab4_utils.py` 상수 (`TAB4_SELECTOR`, `TAB4_GRID_ROOT`)

### RealGrid 예외 규칙(빈칸 정상 항목)이 바뀌었을 때
- `measin_constants.py` 한 곳만 수정하면 eco_input/eco_check 양쪽 자동 반영

### 선행문자 제거(×, ✕, 쉼표) 로직이 바뀌었을 때
- `data_utils.py`의 `clean_leading_mark()` 한 곳만 수정

### 엑셀 시트명이 바뀌었을 때
- 대기: `eco_input.py` → `AIR_SHEET_ANALY`, `AIR_SHEET_RECORD`, `AIR_SHEET_COVER`
- 수질: `eco_input.py` → `WATER_SHEET_ANALY`, `WATER_SHEET_RECORD`

### eco_input.py에 input() 질문 순서가 바뀌었을 때
- **반드시** `eco_input_gui.py`의 `_build_answers_air()` / `_build_answers_water()` 도 같이 수정

### 새 매체(예: 소음)를 추가할 때
1. `eco_input.py` 상단에 상수 추가 (`FIELD_URL_XXX`, `NAS_BASE_XXX` 등)
2. `_main_xxx()` 함수 추가
3. `main()`에 분기 추가
4. `eco_input_gui.py` 매체 선택 라디오버튼 추가
5. `_build_answers_xxx()` 추가
