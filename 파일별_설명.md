### `eco_input.py` (약 2,740줄)
**역할:** 측정인.kr 사이트에 측정 데이터를 자동으로 입력하는 메인 로직

#### 진입점
```
main()
  ├─ 매체 선택 (1=대기 / 2=수질)
  ├─ _main_air()   ← 대기 흐름
  └─ _main_water() ← 수질 흐름
```

#### 대기 흐름 `_main_air()`
| 단계 | 함수 | 설명 |
|------|------|------|
| 시료 목록 수집 | `extract_samples_from_nas()` / `parse_sample_input()` | 팀+날짜 자동추출 또는 직접입력 |
| 탭1 | `fill_tab1()` | 현장측정정보 입력 |
| 탭2 | `fill_tab2()`, `rg2_fill_measure_grid_api()` | 시료채취/측정정보 + 리얼그리드 |
| 탭2 PDF | `export_pdf_from_excel()` | 측정기록부 PDF 생성 후 업로드 |
| 백데이터 | `export_backdata_moist_thc()` | 수분 CSV + THC CSV/FID 저장 |
| 탭4 | `fill_tab4_dates()`, `fill_tab4_grid_only()` | 측정분석결과 입력 |
| 탭4 PDF | `make_tab4_pdfs()` | 대기 시트 PDF 생성/업로드/최종본 저장 |

#### 수질 흐름 `_main_water()`
탭1/2/백데이터 없음. **탭4 PDF 업로드 + 저장만** 수행
```
탭4 클릭 → make_tab4_pdfs_water() → upload_tab4_pdfs() → tab4_comp_save()
```

#### 주요 상수 (파일 상단 74~105줄)
```python
# 대기
FIELD_URL        = "https://측정인.kr/ms/field_outair.do"
NAS_BASE         = r"\\...\2.성적서"
NAS_DIRS         = ["0 0.입력중", "0 1.완료", ...]
PDF_BASE_DIR     = r"\\...\0.PDF\2.대기pdf"
FINAL_DONE_DIR   = r"\\...\0 5.최종완료"
AIR_FILE_BTN1/2  = "#newFile1" / "#newFile2"
AIR_SHEET_ANALY  = "대기시료채취 및 분석일지"
AIR_SHEET_RECORD = "대기측정기록부"
AIR_SHEET_COVER  = "시험항목날짜서명"

# 수질
FIELD_URL_WATER    = "https://측정인.kr/ms/field_outwater.do"
NAS_BASE_WATER     = r"\\...\14.수질성적서"    ← YYYY년\업체명\ 하위
PDF_BASE_DIR_WATER = r"\\...\0.PDF\1.수질pdf"  ← YYYY년\업체명\ 하위
WATER_FILE_BTN1/2  = "#anzeFile1" / "#anzeFile2"
WATER_SHEET_ANALY  = "분석일지"
WATER_SHEET_RECORD = "대행기록부"
```

#### 자주 수정할 함수
| 함수 | 수정 시점 |
|------|----------|
| 상수 블록 | NAS 경로 변경, 사이트 URL 변경, 셀렉터 변경 시 |
| `make_tab4_pdfs()` | 대기 엑셀 시트명 변경 시 |
| `make_tab4_pdfs_water()` | 수질 엑셀 시트명 변경 시 |
| `build_final_paths()` | 대기 PDF 저장 경로 구조 변경 시 |
| `build_final_path_water()` | 수질 PDF 저장 경로 구조 변경 시 |
| `upload_tab4_pdfs()` | 셀렉터 파라미터로 받음 → 대기/수질 둘 다 이 함수 사용 |

#### 주요 흐름
```

→ 엑셀 데이터 읽기(read_excel_for_sample)
→ 비교(build_realgird_compare_rows) → 결과 출력/저장
```

| 함수 | 수정 시점 |
|------|----------|
| `read_site_data()` | 사이트 HTML 구조 변경 시 |
| `build_excel_realgird_expected()` | 엑셀 셀 위치/시트명 변경 시 |
| `compare_scalar()` / `compare_list()` | 비교 로직/허용 오차 변경 시 |

### `report_check.py` (약 1,435줄)
**역할:** 성적서 엑셀 파일 내용 검토 (시간 겹침, 농도 한계값, 장비 누락 등)

#### 주요 흐름
```
main(sample_list, user_name)
→ 각 시료별 엑셀 읽기 → 여러 검사 수행 → 결과 엑셀 출력
```

#### 주요 검사 항목
- `check_group_time_alignment()` : 채취 시간 그룹 정합성
- `check_overlap_events()` : 시료채취 시간 겹침
- `build_conc_limit_checks()` : 농도 한계값 초과 여부
- `check_device_missing_but_measured()` : 측정은 했는데 장비 기록 없음

## 수정 시 체크리스트

### NAS 경로가 바뀌었을 때
- `eco_input.py` 상단 상수 블록 (`NAS_BASE`, `NAS_BASE_WATER` 등)
- `eco_check.py` 상단 상수 블록

### 사이트 셀렉터가 바뀌었을 때
- `eco_input.py` 상수 블록 (`AIR_FILE_BTN1/2`, `WATER_FILE_BTN1/2`, `TAB4_SELECTOR` 등)
- `eco_check.py` 내 해당 셀렉터 상수

### 엑셀 시트명이 바뀌었을 때
- 대기: `eco_input.py` → `AIR_SHEET_ANALY`, `AIR_SHEET_RECORD`, `AIR_SHEET_COVER`
- 수질: `eco_input.py` → `WATER_SHEET_ANALY`, `WATER_SHEET_RECORD`

### eco_input.py에 input() 질문 순서가 바뀌었을 때
- **반드시** `eco_input_gui.py`의 `_build_answers_air()` / `_build_answers_water()` 도 같이 수정

### 새 매체(예: 소음)를 추가할 때
1. `eco_input.py` 상단에 상수 추가 (`FIELD_URL_XXX`, `NAS_BASE_XXX` 등)
2. `_main_xxx()` 함수 추가
3. `main()`에 분기 추가
4. `eco_input_gui.py` 매체 선택 라디오버튼 추가
5. `_build_answers_xxx()` 추가
