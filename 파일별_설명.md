# 파일별 설명 및 유지보수 가이드

> 마지막 업데이트: 2026-04-23
> 대상 독자: 개발자, 유지보수 담당자, 인수인계 문서를 보는 실무자

이 문서는 `readme.md`보다 더 기술적인 문서다.
목적은 다음과 같다.

- 각 파일이 정확히 어떤 책임을 가지는지 빠르게 파악
- 수정하려는 기능이 어느 파일에 있는지 찾기
- 한 파일을 수정할 때 같이 확인해야 할 연관 파일을 놓치지 않기

---

## 1. 최상위 구조

이 코드베이스는 크게 4개 영역으로 나뉜다.

1. 실행 진입점 `.bat`
2. GUI 파일
3. 핵심 업무 로직 파일
4. 공통 유틸리티 파일

추가로 아래 영역도 함께 본다.

- 초기 설치 폴더 `0.처음사용시`
- 문서 파일들 (`readme.md`, `버전노트.txt`, `모듈화_분석.md` 등)
- `__pycache__` 폴더

## 2. 배치 파일과 실제 실행 대상 매핑

| 배치 파일 | 실제 실행 파일 | 비고 |
|------|------|------|
| `1.측정인 자동입력 V2.7.bat` | `eco_input_gui.py` | 대기/수질 자동입력, 백데이터, 탭4 PDF |
| `2.측정인 검토 V2.3.bat` | `eco_check_gui.py` | 측정인 사이트와 엑셀 비교 |
| `3.발송대장 검토 V1.4.bat` | `receipt.py` | GUI와 로직 통합 |
| `4.종합 검토 V1.5.bat` | `dash.py` | GUI와 로직 통합 |
| `5.성적서 검토 V2.5.bat` | `report_check_gui.py` | GUI 래퍼 |
| `6.차량운행일지 검토 V1.6.bat` | `Vehicle_operation_log.py` | GUI와 로직 통합 |
| `7.PDF 생성 V1.1.bat` | `tab4_pdf_final_gui.py` | GUI와 로직 통합 |

배치 파일은 대부분 아래 3줄 구조다.

```bat
@echo off
chcp 65001 >nul
start "" pythonw "%~dp0...py"
```

의미는 다음과 같다.

- 한글 깨짐 방지를 위해 UTF-8 코드페이지 사용
- 현재 폴더 기준으로 GUI Python 파일 실행
- 콘솔창 없이 GUI만 띄움

배치 파일 버전을 바꾸면 다음도 같이 확인해야 한다.

- `버전노트.txt`
- 실제 배치 파일명
- 설명 문서의 버전 표기

## 3. GUI 계층

## 3-1. `eco_input_gui.py`

역할:

- `eco_input.py`를 GUI로 감싼 메인 입력 UI
- 대기/수질 모드 전환
- 시료번호 직접 입력 또는 파일 드래그앤드롭
- 실행 옵션 체크박스 관리
- 백그라운드 실행과 로그 표시

특징:

- `tkinterdnd2`가 있으면 DnD 활성화
- DnD import는 `pythoncom/win32com`보다 먼저 해야 충돌을 줄일 수 있음
- 내부적으로는 `eco_input.main()`의 `input()` 흐름을 가로채는 구조

같이 봐야 하는 파일:

- `eco_input.py`
- `gui_common.py`
- `worker_utils.py`
- `data_utils.py`

수정 시 주의:

- `eco_input.py`의 질문 순서가 바뀌면 GUI의 answer 배열도 같이 수정
- DnD 동작을 손대면 import 순서와 fallback 동작 확인 필요

## 4. 핵심 업무 로직 계층

## 4-1. `eco_input.py`

역할:

- 이 코드베이스에서 가장 큰 메인 오케스트레이션 파일
- 대기와 수질 입력 흐름을 모두 담당

진입점:

- `main()`

주요 분기:

- `_main_air()`
- `_main_water()`

대기 처리 흐름 요약:

1. 작업 모드/로그인/옵션 입력
2. 시료 목록 수집
3. 사이트 로그인 및 날짜 검색
4. 탭1 입력
5. 탭2 입력
6. 백데이터 생성
7. 탭2 PDF 생성/업로드
8. 탭4 입력 및 PDF 처리
9. 저장 후 목록 복귀

수질 처리 흐름 요약:

1. 로그인
2. 수질 시료번호 입력
3. 상세 페이지 진입
4. 탭4만 처리
5. PDF 업로드 후 저장

자주 수정하는 부분:

- 상단 NAS/PDF/URL 상수
- 탭1/탭2 셀렉터
- 탭4 업로드 셀렉터
- PDF 저장 경로
- 수질/대기 시트명

같이 수정될 수 있는 파일:

- `eco_input_gui.py`

## 4-3. `report_check.py`

역할:

- 성적서 파일의 자체 품질 검토

주요 검사 예:

- 시간 겹침
- 그룹 정합성
- 농도 한계값
- 장비 누락
- 시트 구성 이상

최근 반영 사항:

- 입자상 채취시간이 수분량자동측정기/가스자동측정기 시간과 겹치지 않는지 검사
- 입자상보다 수분량자동측정기/가스자동측정기를 먼저 수행했는지 검사
- 가스자동측정기 시간은 `황산화물`, `질소산화물`, `일산화탄소` 행의 시간으로 단일 이벤트화
- 위 3개 항목이 둘 이상 측정되면 채취시간 일치 여부 검사
- `sample-2`에 발송대장 월별 파일에서 `가스미터 전/후`, `voc 전/후` 값을 채움

같이 수정될 수 있는 파일:

- `report_check_gui.py`
- `excel_utils.py`
- `data_utils.py`
- `format_utils.py`

## 4-4. `backdata_utils.py`

역할:

- 수분량 CSV, THC CSV/FID를 뽑아내는 전용 모듈
- 원래 `eco_input.py`에 있던 백데이터 로직을 분리한 파일

핵심 함수:

- `export_backdata_moist_thc()`
- `export_csv_display_as_is()`
- `export_fid_by_excel_copy()`

중요 동작:

- NAS 안전 저장 재시도
- Excel 표시값 그대로 복사
- 클립보드 polling 기반 복사
- 항목별 종료시간 전 생성 스킵

현재 종료시간 스킵 규칙:

- 수분: `입력!B23` 시작시간 + 5분
- THC: `입력(분석값)!K64` 기준
  - FID 모드: 해당 시각까지 대기
  - PF 모드: +3분까지 대기

생성 전이면 다음 로그가 뜬다.

- `⚠ 수분 백데이터 생성 스킵 ...`
- `⚠ THC 백데이터 생성 스킵 ...`

같이 수정될 수 있는 파일:

- `receipt.py`
- `eco_input.py`
- `excel_utils.py`

## 10. 수정할 때 자주 놓치는 것

1. `eco_input.py` 질문 순서 변경 후 `eco_input_gui.py` 미수정
2. 배치 파일 버전 올리고 `버전노트.txt`나 파일명 미수정
3. NAS 경로를 한 파일만 바꾸고 다른 모듈은 그대로 둠
4. 시트명 변경을 `eco_input.py`만 바꾸고 `backdata_utils.py`, `pdf_utils.py`, `report_check.py`는 안 바꿈
5. selector 변경을 `eco_input.py`에만 넣고 `eco_check.py` 공통 상수는 안 바꿈
6. 비산먼지 규칙 변경 시 `file_utils.py`와 비교 예외 규칙을 같이 안 봄
7. DnD 관련 import 순서를 건드려서 `tkinterdnd2` 충돌 유발
