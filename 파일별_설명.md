# 파일별 설명 및 유지보수 가이드

> 마지막 업데이트: 2026-05-30 (`3.py` 폴더 구조)
> 대상 독자: 개발자, 유지보수 담당자, 인수인계 문서를 보는 실무자

이 문서는 `readme.md`보다 더 기술적인 문서다.
목적은 다음과 같다.

- 각 파일이 정확히 어떤 책임을 가지는지 빠르게 파악
- 수정하려는 기능이 어느 파일에 있는지 찾기
- 한 파일을 수정할 때 같이 확인해야 할 연관 파일을 놓치지 않기

---

## 1. 최상위 구조

이 코드베이스는 크게 4개 영역으로 나뉜다.

1. 프로그램 루트: `2.검토 및 입력프로그램.pyw` (런처), `config.ini`
2. `3.py\` 폴더: GUI·핵심 로직·공통 유틸 Python 전부

추가로 아래 영역도 함께 본다.

- 초기 설치 폴더 `0.처음사용시`
- 문서 파일들 (`readme.md`, `버전노트.txt`, `모듈화_분석.md` 등)
- `__pycache__` 폴더

## 2. 통합 런처와 실제 실행 대상 매핑

| 런처 버튼(표시명) | 실제 실행 파일 (`3.py\` 기준) | 비고 |
|------|------|------|
| 1. 측정인 자동입력 | `eco_input_gui.py` | 대기/수질 자동입력, 백데이터, 탭4 PDF |
| 2. 측정인 검토 | `eco_check_gui.py` | 측정인 사이트와 엑셀 비교 |
| 3. 발송대장 검토 | `receipt.py` | GUI와 로직 통합 |
| 4. 종합 검토 | `dash.py` | GUI와 로직 통합 |
| 5. 성적서 검토 | `report_check_gui.py` | GUI 래퍼 |
| 6. 차량운행일지 검토 | `Vehicle_operation_log.py` | GUI와 로직 통합 |
| 7. PDF 생성 | `tab4_pdf_final_gui.py` | GUI와 로직 통합 |

진입점: 프로그램 루트 **`2.검토 및 입력프로그램.pyw`** → **`3.py\`** 안의 스크립트 실행

- `.pyw` 더블클릭 → 런처만 콘솔 없이 실행
- 하위 GUI: `subprocess` + `pythonw` + `CREATE_NO_WINDOW`, **`cwd=3.py`**
- `config.ini`는 **루트**에 두고, `3.py\config.py`가 상위 폴더의 ini를 읽음

예전 `1.~7.` 번호 `.bat` 파일은 삭제됨. **`0.처음사용시\*.bat`** 만 설치용으로 유지.

버전·변경 이력은 파일명이 아니라 **`버전노트.txt`** 에만 기록한다. (런처 변경 → `0.검토 및 입력프로그램`, 도구 로직 변경 → `1.측정인 자동입력 Vx.y` 형식)

## 3-1. `eco_input_gui.py`

역할:

- `eco_input.py`를 GUI로 감싼 메인 입력 UI
- 대기/수질 매체 전환, 시료번호·파일 드래그앤드롭, 옵션 체크박스, 백그라운드 실행·로그

특징:

- `tkinterdnd2`가 있으면 DnD 활성화 (import는 `eco_input`보다 **먼저** — COM 충돌 방지)
- `eco_input`은 worker 스레드에서 **lazy import**
- 실행: `_build_answers_air()` / `_build_answers_water()` → dict → `_main_air()` / `_main_water()` **키워드 직접 호출** (`main()`, `fake_input`, `builtins.input` 미사용)
- 창 기본 `860×820`, min 높이 `720`, 시작·취소 버튼 가운데 정렬

대기 GUI (`_build_answers_air`):

| 체크박스 | dict 키 → `_main_air` 인자 |
|----------|---------------------------|
| 현장측정정보 (탭1) | `do_tab1` |
| 백데이터 | `do_backdata` |
| 시료채취/측정 (탭2) | `do_tab2` |
| 탭2 PDF | `do_pdf_upload` (탭2 OFF면 False) |
| 측정분석결과 (탭4) | `do_tab4` |
| 탭4 PDF | `do_pdf_final` (탭4 OFF면 False) |

- 작업 `job=2`(백데이터만): 위 사이트·PDF 플래그는 모두 False, `do_backdata=True`만 유지
- 모드1: `raw_samples` / 모드2: `team_no`, `date_str`

수질 GUI (`_build_answers_water`):

| 체크박스 | dict 키 → `_main_water` 인자 |
|----------|-----------------------------|
| 시험의뢰정보 (탭2) | `do_tab2` |
| 탭4 | `do_tab4` |
| 탭4 PDF | `do_pdf_final` (탭4 OFF면 False) |

- 파일 추가·실행 전 NAS 확인: `config.WATER_REPORT_INPUT` (`0.입력중`)
- 로그인: 항상 `login_id`, `login_pw` 필수

같이 봐야 하는 파일:

- `eco_input.py` (`_main_air`, `_main_water` 시그니처)
- `gui_common.py`
- `data_utils.py` (시료번호 `A`/`W`, 날짜)
- `config.py` (`WATER_REPORT_INPUT`, `PDF_WATER`, `WATER_RECORD_PDF_DIR`, `WATER_TAB4_*`)

수정 시 주의:

- 옵션 추가·삭제 시: GUI dict, `_build_answers_*`, `_main_*()` 인자, (콘솔용) `ask_yesno` 분기를 함께 맞출 것
- 체크박스 **화면 배치만** 바꿀 때는 bool 매핑만 유지하면 됨 (콘솔 질문 순서와 무관)
- DnD·import 순서 변경 시 fallback 동작 확인

## 4-1. `eco_input.py`

역할:

- 이 코드베이스에서 가장 큰 메인 오케스트레이션 파일
- 대기와 수질 입력 흐름을 모두 담당

진입점:

- `main()` — 매체 선택 후 `_main_air()` 또는 `_main_water()` (콘솔: `input()` / `ask_yesno()`)

주요 분기:

- `_main_air(...)` — `job` 등 키워드가 넘어오면 **gui_mode** (GUI 종료 시 `input("엔터...")` 생략)
- `_main_water(...)` — `login_id` 등 키워드가 넘어오면 **gui_mode** (`input("엔터...")` 생략, `lost sys.stdin` 방지)

대기 처리 흐름 요약:

1. 작업 모드/로그인/옵션 입력
2. 시료 목록 수집
3. 사이트 로그인 및 날짜 검색
4. 탭1 입력 (`fill_tab1`) — 입력 시트 **F10** → `#edit_meas_purpose` (1=`SELF` 자가측정용, 2=`CF` 참고용). 인력·차량·장비는 `format_utils` 슬래시 규칙으로 Select2 등록 (`readme.md` §4-1-2). `parse_measuring_record`·`eco_check` 비교와 동일
5. 탭2 입력
6. 백데이터 생성
7. 탭2 PDF 생성/업로드
8. 탭4 입력 및 PDF 처리
9. 저장 후 목록 복귀

수질 처리 흐름 요약:

1. `field_water.do` 로그인 → **시료번호만** 목록 검색 (`measin_utils.open_sample_detail`, `date_str=None` — 날짜 `search_date` 없음)
2. 상세 진입 (`WATER_DETAIL_WAIT_SEL` = `#samp_vesl_code`, NAS: `find_sample_file_in_water_nas` → `WATER_REPORT_INPUT`)
3. 탭2(옵션): PE / 1 / 1 / 폭기조 → `#btnSaveMsFieldDoc` (확인 2회). 탭4만·탭2 미완료면 탭2 선행
4. 탭4(옵션): `open_water_tab4()` → `read_water_tab4_from_macro_xlsm()` → `fill_water_tab4_dates()` → `fill_water_tab4_grid()`
5. PDF(옵션, `do_pdf_final`):
   - `make_tab4_pdfs_water()` — 시트 `분석일지`·`대행기록부` PDF → 병합·저장·대행만 복사
   - `upload_tab4_pdfs(..., f1_sel=#anzeFile1, f2_sel=#anzeFile2)`
   - `save_water_tab4_complete()` → `#btnCompSave` (확인 2회)
6. PDF OFF: `save_water_tab4_temp()` → `#btnTempSave`
7. `back_to_list()` → 다음 시료

수질 PDF·경로 함수 (`eco_input.py`):

| 함수 | 역할 |
|------|------|
| `read_water_record_d3()` | 병합 PDF 하위 폴더명: **대행기록부** 시트 **D3**, `\s+` 제거 |

자주 수정하는 부분:

## 10. 수정할 때 자주 놓치는 것

1. `eco_input.py`에 옵션·인자 추가 후 `eco_input_gui.py`의 `_build_answers_*` / `_run_automation` 미반영 (검토 도구 `eco_check_gui`는 여전히 answer 배열 순서 의존)
2. 코드만 고치고 `버전노트.txt` 해당 도구 버전 줄 미추가
3. NAS 경로를 한 파일만 바꾸고 다른 모듈은 그대로 둠
4. 시트명 변경을 `eco_input.py`만 바꾸고 `backdata_utils.py`, `pdf_utils.py`, `report_check.py`는 안 바꿈
5. selector 변경을 `eco_input.py`에만 넣고 `eco_check.py` 공통 상수는 안 바꿈
6. 비산먼지 규칙 변경 시 `file_utils.py`와 비교 예외 규칙을 같이 안 봄
7. DnD 관련 import 순서를 건드려서 `tkinterdnd2` 충돌 유발
